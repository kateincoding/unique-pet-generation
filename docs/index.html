Unique Pet Generation

Phase 1: Baseline de Extraccion de Rasgos Faciales — Katherine Soto, Mayo 2026

1. Objetivo del Proyecto

El objetivo final de este proyecto es generar mascotas que se parezcan a sus humanos: que compartan color de ojos, tono de pelo, estructura facial y otros rasgos. Para lograr esto, el primer paso (Phase 1) es construir un modelo baseline que extraiga rasgos faciales de fotos humanas y los codifique en un vector de embedding que luego se usara para condicionar la generacion de mascotas.

2. Marco Teorico

2.1 Transfer Learning con ResNet-18

ResNet-18 es una red neuronal convolucional de 18 capas con conexiones residuales (skip connections) que resuelven el problema del vanishing gradient en redes profundas. Al usar un modelo pre-entrenado en ImageNet (1.2M imagenes, 1000 clases), aprovecho representaciones visuales ya aprendidas como bordes, texturas y formas, y las adapto a mi tarea especifica de clasificacion de atributos faciales.

2.2 Multi-Task Learning

En lugar de entrenar 15 modelos separados (uno por atributo), use un unico modelo con multiples salidas. Esto tiene dos ventajas: las tareas comparten representaciones en las capas intermedias (lo que mejora la generalizacion), y fuerza al embedding bottleneck a capturar informacion relevante para todos los atributos simultaneamente.

2.3 Embedding Bottleneck

Entre el backbone (512 dimensiones) y el clasificador, inserte una capa de proyeccion que reduce a 256 dimensiones. Este bottleneck obliga al modelo a comprimir la informacion facial en una representacion densa y semantica. Este vector de 256 dimensiones es el puente hacia Phase 2: sera el input de condicionamiento para el modelo generativo de mascotas.

2.4 Dataset: CelebA

Inicialmente considere WIDER FACE, pero ese dataset solo contiene bounding boxes para deteccion de caras — no tiene atributos faciales. CelebA es la opcion correcta: 202,599 imagenes de rostros alineados, cada uno anotado con 40 atributos binarios incluyendo color de pelo, forma de cara y caracteristicas de ojos y nariz. Use la version de Kaggle (jessicali9530/celeba-dataset).

3. Arquitectura del Modelo

Input (224x224 RGB) | ResNet-18 pretrained (trunk) ──── 512-dim feature vector | Linear(512, 256) + ReLU + Dropout(0.3) ──── 256-dim EMBEDDING | Linear(256, 15) ──── attribute logits

Decisiones de diseno

Componente Eleccion Razon
Backbone ResNet-18 Balance entre capacidad y velocidad de entrenamiento para un baseline. Entrena rapido (~1-2h en MPS) y produce features solidas
Embedding dim 256 Suficiente para capturar semantica facial sin overfitting. Vector pequeno = representacion de alto nivel, no detalles pixel-level
Dropout 0.3 Regularizacion moderada en el embedding para prevenir co-adaptacion de features
Num. atributos 15 de 40 Seleccione los atributos que mapean a rasgos de mascotas: color de pelo → pelaje, forma de cara → cara de mascota
Deteccion facial MTCNN (pretrained) Solo para inferencia en fotos arbitrarias. CelebA ya viene con imagenes alineadas

4. Decisiones de Entrenamiento

Parametro Valor Justificacion
Loss function BCEWithLogitsLoss + pos_weight Combina sigmoid + BCE en una operacion numericamente estable. pos_weight compensa el desbalance de clases (ej: Gray_Hair solo ~4% positivo)
Optimizer AdamW (lr=1e-3, wd=1e-4) AdamW con weight decay desacoplado. lr=1e-3 es agresivo para heads nuevos pero funciona bien con backbone congelado
Scheduler CosineAnnealingLR Reduce lr suavemente hacia 0. Mejor que StepLR para convergencia final
Backbone freeze 5 epochs Primero entreno solo embedding + classifier para que aprendan antes de ajustar backbone. Despues fine-tune con lr*0.1 para no destruir features pre-entrenadas
Gradient clipping max_norm=1.0 Previene explosion de gradientes durante fine-tuning del backbone
Early stopping patience=5 Detiene training si val mAP no mejora por 5 epochs. Previene overfitting
Batch size 64 Cabe en memoria de Apple Silicon. Suficiente para estimaciones estables de gradiente
Data augmentation RandomResizedCrop, HFlip, ColorJitter, GaussNoise Simula variaciones reales de fotos: diferentes encuadres, iluminacion, ruido de camara
Device MPS (Apple Silicon) GPU integrada del Mac. MTCNN forzado a CPU por incompatibilidad con adaptive pooling de MPS

7. Analisis y Observaciones

Lo que funciono bien

  • Atributos de estructura facial (Bushy_Eyebrows, High_Cheekbones, Oval_Face, Chubby) se clasificaron correctamente con probabilidades altas (>0.59)
  • El modelo descarto correctamente atributos irrelevantes como Blond_Hair (0.001) y Pale_Skin (0.020)
  • El embedding de 256 dimensiones captura representaciones semanticas utiles

Areas por mejorar

  • Color de pelo: ninguno de los 4 atributos de color supero el threshold de 0.5. Black_Hair quedo en 0.427, lo cual es cercano pero no suficiente. Esto puede deberse a que CelebA tiene sesgo hacia ciertos tonos y la iluminacion de webcam afecta la percepcion del color
  • Textura de pelo: Straight_Hair (0.163) y Wavy_Hair (0.045) no lograron clasificar correctamente. Esto sugiere que el modelo necesita mejor preprocesamiento para diferenciar texturas capilares
  • Mejora de datos: la imagen de webcam tiene condiciones distintas a CelebA (iluminacion, angulo, resolucion). Un paso de normalizacion o domain adaptation podria ayudar

8. Siguientes Pasos (Phase 2)

  • Mejorar el preprocesamiento de imagenes para capturar mejor el color y textura del pelo
  • Experimentar con backbones mas grandes (EfficientNet-B0) o agregar atributos continuos
  • Colectar o curar dataset de mascotas diversas con variedad de razas, colores y rasgos
  • Entrenar modelo generativo condicional (diffusion model o StyleGAN) que reciba el embedding de 256-dim como condicionamiento
  • Mapping semantico: "pelo oscuro + cejas pobladas + cara redonda" → mascota con pelaje oscuro, expresion marcada, cara redondeada
